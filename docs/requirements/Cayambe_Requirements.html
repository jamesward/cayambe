Systemic Requirements

  • Catalog Product Data should be cached in memory so as to improve performance
  • Products and categories should be easily extensible
  • The checkout process should be easily extensible
  • The shopping cart should handle multiple payment methods
  • The shopping cart should handle multiple shipping locations
  • The application should pass off orders to a fulfillment system
  • The UI should be easily modified without exposing logic code.
  • Extensive data should be gathered from a customer.

Non-Functional Requirements

  • The system must handle multiple storefronts within one running instance (ie. Multiple customers can have their store run through one instance of the shopping cart).
  • A default UI implementation must be provided.
  • The application must be able to bulk load inventory from other systems in XML format.
  • The shopping cart must be designed for I18N.
