Instructions for using Cayambe CVS
 
First!!!
These instructions are written somewhat general, but also,
somewhat
specific to using cvs in unix.
You must set your CVS_RSH environement variable to ssh
In unix under bash or ksh do this:
export CVS_RSH=ssh
This tells cvs to use ssh as the transport agent.
You also must have your CVSROOT env var set to
:ext:USERNAME@cvs.cayambe.sourceforge.net:/cvsroot/cayambe
In unix under bash or ksh do this:
export
CVSROOT=:ext:USERNAME@cvs.cayambe.sourceforge.net:/cvsroot/cayambe
This tells cvs that your are using the server
cvs.cayambe.sourceforge.net and the repository /cvsroot/cayambe
*** Of course you will need to replace USERNAME with your s.f.
username.


Getting Cayambe Files
Once you have your enviroment vars correct, it is pretty easy to
get the
files. This is called checkout. All you have to do is:
cvs checkout MODULENAME
*** Replace MODULENAME with the module you want to get. ie. docs
or core
This will get all of the cvs files and put them on your local
computer
in the current directory.


Making Changes to files
All you have to do is edit your files on your local computer.
When you
are satisfied with the changes and want to move them to the cvs
server,
you will need to merge your changes and then commit your files.
Like this:
cvs update
cvs commit 
The update will merge your changes with any on the cvs server.
The commit will save your changes to the server.
Commit will ask you to give a description of what changes you
made.
Please be as descriptive as possible!


Adding new files to CVS
If you have just created a file you would like to add to CVS,
you can do
so by using the add command. Like:
cvs add blah.java

Step 3:

Set Configs and Deploy Application

Important Steps

1. Set your environmental variables:

JAVA_HOME


J2EE_HOME


STRUTS_HOME


ANT_HOME

2. Use the build.sh script to make the build.

Run build in the {Project_Home}/core/src directory.

3. Install the mySQL driver

Place jar in {JBOSS_HOME}/jboss/lib/ext

4. Setup your Datasource in Jboss

See example configs at end of doc

5. Run the mysql.sql and test_data.sql script

Run these in the mysql database.

6. Deploy the application.

Place the cayambe.ear in {JBOSS_HOME}/jboss/deploy
