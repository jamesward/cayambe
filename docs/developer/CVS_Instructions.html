Instructions for using Cayambe CVS
First!!!
These instructions are written somewhat general, but also, somewhat specific to using cvs in unix.
You must set your CVS_RSH environement variable to ssh
In unix under bash or ksh do this:
export CVS_RSH=ssh
This tells cvs to use ssh as the transport agent.
You also must have your CVSROOT env var set to :ext:USERNAME@cvs.cayambe.sourceforge.net:/cvsroot/cayambe
In unix under bash or ksh do this:
export CVSROOT=:ext:USERNAME@cvs.cayambe.sourceforge.net:/cvsroot/cayambe
This tells cvs that your are using the server cvs.cayambe.sourceforge.net and the repository /cvsroot/cayambe
*** Of course you will need to replace USERNAME with your s.f. username.


Windows Instructiosn (Thanks to Mike Broadbear)
In order to download the catambe code from sourceforge, you must obtain a CVS client and an SSH client. The easiest way to accomplish this is with Cygwin. Cygwin is a Unix emulation layer for Windows with many useful features.

Download Cygwin from:
http://www.cygwin.com

**Make sure to select the 'cvs' utility and the 'openssh' utilities from the application list.

Open Cygwin.

Set the proper environment variables.
At the bash command prompt, type:
export CVS_RSH=ssh
export CVSROOT=:ext:USERNAME@cvs.cayambe.sourceforge.net:/cvsroot/cayambe

In order to establish the proper subdirectories on the remote CVS servers, log in with each of the following:
ssh -l [username] cayambe.sourceforge.net
ssh [username]@cvs.cayambe.sf.net

Change the current directory to the directory that will serve as the cayambe home.
Ex:
cd /cygdrive/c/cayambe

Check out files! Type:
cvs co core - download the core cayambe files
cvs co docs - download the cayambe documentation

Getting Cayambe Files
Once you have your enviroment vars correct, it is pretty easy to get the files. This is called checkout. All you have to do is:
cvs checkout MODULENAME
*** Replace MODULENAME with the module you want to get. ie. docs or core
This will get all of the cvs files and put them on your local computer in the current directory.


Making Changes to files
All you have to do is edit your files on your local computer. When you are satisfied with the changes and want to move them to the cvs server, you will need to merge your changes and then commit your files.
Like this:
cvs update
cvs commit
The update will merge your changes with any on the cvs server.
The commit will save your changes to the server.
Commit will ask you to give a description of what changes you made. Please be as descriptive as possible!


Adding new files to CVS
If you have just created a file you would like to add to CVS, you can do so by using the add command. Like:
cvs add blah.java
